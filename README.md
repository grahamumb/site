# Personal website

A tiny, zero-dependency static site. Vanilla HTML/CSS/JS. Posts are markdown
files; a GitHub Action rebuilds and deploys to GitHub Pages on every push.

## Writing a post

1. Add a file to `posts/` named `YYYY-MM-DD-some-title.md`.
   - The **date prefix** controls ordering (newest first) and is shown on the post.
   - Start the file with a `# Title` line — that first heading becomes the title
     on the falling rectangle and in the post.
2. Commit and push to `main`.
3. The Action rebuilds the site and redeploys. The new title appears as a
   falling rectangle on the Home page and is clickable to read.

That's the whole workflow. No other steps.

### Markdown supported

Plain text, paragraphs, single-newline line breaks, `#`/`##`/`###` headings,
`![alt](src)` images, `[text](url)` links, `**bold**`, and `*italic*`.
To extend it, edit the converter at the top of `build.js`.

## Editing the About page

Edit `about.md`. Same markdown rules.

## Project layout

```
posts/                     your markdown posts (the only thing you edit day to day)
about.md                   the About Me page
src/                       static site source (copied verbatim into dist/)
  index.html  styles.css  app.js  physics.js
build.js                   converts md -> html, writes the manifest, builds dist/
.github/workflows/deploy.yml   builds + deploys on push
```

`dist/` is generated by the build and is **not** committed.

## Local preview

```sh
node build.js
cd dist && python3 -m http.server 8000
# open http://localhost:8000
```

## One-time GitHub setup

In the repo on GitHub: **Settings → Pages → Build and deployment → Source =
GitHub Actions**. After the first push to `main`, the Action deploys the site.
